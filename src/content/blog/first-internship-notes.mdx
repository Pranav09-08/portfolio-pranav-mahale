## Start with the user problem

At first, I focused too much on writing code quickly. The real improvement came when I spent more time understanding the user workflow and pain points.

## Keep pull requests small

Smaller pull requests got reviewed faster, had fewer bugs, and made discussions clearer. I now aim for scoped changes with strong commit messages.

## Performance matters earlier than expected

Even simple pages felt slow on weaker devices when images and scripts were not optimized. Monitoring Core Web Vitals early made a visible difference.
